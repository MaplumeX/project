# ADD & INC 指令功能测试集

本测试在 TEC-8 实验箱上验证 `ADD Rd, Rs` 与 `INC Rd` 两条指令的数据通路执行结果。
控制器源码见 `pipelined_hardwired_controller.v`，指令定义见 `指令格式.md`，执行流程见 `流程图.md`。

> 操作前必读：`测试操作注意事项.md`（陷阱 #1「模式锁存两步法」、陷阱 #2「用灯组合判断 ST0 阶段」）与 `TEC-8寄存器读写时序说明.md`。

---

## 0. 测试目的

- 验证 `ADD R0, R1`：`R0 ← R0 + R1`，结果写回 R0，Z/C 标志随结果更新。
- 验证 `INC R1`：`R1 ← R1 + 1`，结果写回 R1，Z/C 标志随结果更新。
- 顺带暴露 `INC` 分支 ALU 编码疑点（见第 6 节异常处置）。

## 1. 测试程序与初值设计

初值：

| 寄存器 | 初值 | 说明 |
| --- | --- | --- |
| R0 | 0x05 | ADD 目的操作数 |
| R1 | 0x03 | ADD 源操作数 / INC 目的 |

测试程序（连续写入起始地址 0x00 起）：

| 地址 | 机器码 | 指令 | 编码来源（IR7..IR0） | 作用 |
| --- | --- | --- | --- | --- |
| 0x00 | 0x11 | ADD R0, R1 | 0001 00 01 | R0 ← R0 + R1 |
| 0x01 | 0x44 | INC R1 | 0100 01 00 | R1 ← R1 + 1（Rd=R1，IR1-0=XX 取 00） |
| 0x02 | 0xE0 | STOP | 1110 00 00 | 停机 |

预期执行轨迹（取指模式 ST0=1 后，复位 IR=00H=NOP 自然取入第一条）：

```
装 PC=0x00
NOP 周期：IR←MEM[0x00]=0x11(ADD)，PC→0x01
ADD 周期：R0←0x05+0x03=0x08，Z=0,C=0；IR←MEM[0x01]=0x44(INC)，PC→0x02
INC 周期：R1←0x03+1=0x04，Z=0,C=0；IR←MEM[0x02]=0xE0(STOP)，PC→0x03
STOP   ：停机
```

最终预期：R0=0x08，R1=0x04，Z=0，C=0。

## 2. 寄存器初始化（写寄存器模式 100）

目标：R0←0x05，R1←0x03（R2/R3 任意，不影响本测试）。

> 切换到写寄存器模式后，CLUT 后第一次 QD 仅用于让 T3 下降沿把新模式 Q=100 锁存进来（见 `备注.md` 与陷阱 #1）。从第二次 QD 起才真正走写寄存器分支。
> 用 SEL 灯读出"当前拍即将写入哪个寄存器"，再决定 SD 拨什么值（写寄存器全程 DRW 亮）。

| 步骤 | 操作 | 现象（SEL 灯） | 含义 / 备注 |
| --- | --- | --- | --- |
| 2.1 | 拨 `SWC SWB SWA = 100` | —— | 选择写寄存器模式 |
| 2.2 | 按 CLR | SEL 全灭 | ST0=0、Q=000 |
| 2.3 | 拨 SD=任意，按 QD ① | (旧 Q=000 走取指分支，副作用) | 仅锁存 Q←100；ST0 状态以下一步实际 SEL 为准 |
| 2.4 | 观察当前 SEL 灯，按下列对应关系拨 SD 后按 QD，依次写入 4 个寄存器 | —— | 每次按 QD 前 SD 必须拨到位（T3 写入边沿在本次 QD 末尾） |

SEL 灯与寄存器对应表（见 `TEC-8寄存器读写时序说明.md`）：

| SEL3..SEL0 | 目标寄存器 | 本次应拨 SD |
| --- | --- | --- |
| 0011 | R0 | 0x05 |
| 0100 | R1 | 0x03 |
| 1001 | R2 | 任意（如 0x00） |
| 1110 | R3 | 任意（如 0x00） |

按上面的"看 SEL → 拨 SD → 按 QD"循环，直到 R0、R1 都已写入（即出现过 `SEL=0011` 拨 0x05、`SEL=0100` 拨 0x03 两轮），即可停止，不必写满两轮。

## 3. 程序写入存储器（写存储器模式 001）

目标：MEM[0x00]=0x11，MEM[0x01]=0x44，MEM[0x02]=0xE0。

> 严格遵循陷阱 #1「两步法」：切换到 001 后先单独按一次 QD 锁存 Q，再装起始地址。
> 用灯组合判断阶段（陷阱 #2 速查表）：`LAR 亮 / MEMW 灭` = ST0=0 装地址阶段；`MEMW 亮 / ARINC 亮` = ST0=1 写数据阶段。

| 步骤 | 操作 | 预期灯 | 含义 |
| --- | --- | --- | --- |
| 3.1 | 拨 `SWC SWB SWA = 001` | —— | 选择写存储器模式 |
| 3.2 | 按 CLR | —— | ST0=0、Q=000 |
| 3.3 | 按 QD ①（SD 任意） | (锁存 Q←100 的副作用) | 仅锁存 Q←001 |
| 3.4 | 拨 SD=0x00，按 QD ② | (进入装地址或写数据阶段，以灯为准) | 装起始地址 AR←0x00 |
| 3.5 | 拨 SD=0x11，按 QD | `MEMW 亮 / ARINC 亮` | MEM[0x00]←0x11，AR→0x01 |
| 3.6 | 拨 SD=0x44，按 QD | `MEMW 亮 / ARINC 亮` | MEM[0x01]←0x44，AR→0x02 |
| 3.7 | 拨 SD=0xE0，按 QD | `MEMW 亮 / ARINC 亮` | MEM[0x02]←0xE0，AR→0x03 |

若 3.4 按下后看到的是 `MEMW 亮 / ARINC 亮`（已跳过装地址阶段），说明 ST0 已为 1、AR 未被装载。此时需返回 3.2 重新 CLR，再从 3.3 起重做。

## 4. PC 初始化并连续执行（取指模式 000）

目标：从 0x00 开始连续执行 ADD、INC、STOP。

| 步骤 | 操作 | 现象 | 含义 |
| --- | --- | --- | --- |
| 4.1 | 拨 `SWC SWB SWA = 000` | —— | 选择取指模式 |
| 4.2 | 按 CLR | —— | ST0=0、Q=000 |
| 4.3 | 拨 SD=0x00，按 QD ① | `SBUS LPC STOP SHORT` 灯特征 | PC←0x00，ST0→1（停车于此拍） |
| 4.4 | 按 QD ② | （复位 IR=00H=NOP）取第一条 | IR←MEM[0x00]=0x11，PC→0x01 |
| 4.5 | 按 QD ③ | ADD 执行特征灯：`ABUS DRW SHORT CIN LDZ LDC`, S=1001 | R0←0x08；IR←0x44，PC→0x02 |
| 4.6 | 按 QD ④ | INC 执行特征灯：`ABUS DRW SHORT LDZ LDC`（**无 CIN**） | R1←0x04(?)；IR←0xE0，PC→0x03 |
| 4.7 | 按 QD ⑤ | `STOP` 亮 | 停机 |

> 注：取指模式 SWC_SWB_SWA=000 与 CLR 后的缺省 Q=000 一致，4.3 的第一次 QD 既是锁存也是真正取指 ST0=0 的放 PC 拍，不存在两步法延迟问题。

## 5. 结果验证（读寄存器模式 011）

读寄存器为组合读，SEL 一选中即显示内容，无需等 T3（见时序说明第 4 节）。

| 步骤 | 操作 | 现象（SEL 灯） | 数据总线 / 寄存器内容灯 |
| --- | --- | --- | --- |
| 5.1 | 拨 `SWC SWB SWA = 011` | —— | 读寄存器模式 |
| 5.2 | 按 CLR，再按 QD ① 锁存 Q（两步法） | SEL=0001 | 读 R0 / R1 组 |
| 5.3 | 读出 R0 | —— | 期望 0x08 |
| 5.4 | 再按 QD（如需 R2/R3） | SEL=1011 | 读 R2 / R3 组；R1 期望 0x04 |

> 读寄存器具体位与 SEL 连线对应关系以面板实际布线为准。R1 的读取需对照面板上 R1 指示灯位置。

| 寄存器 | 预期值 | 实测值 | 是否通过 |
| --- | --- | --- | --- |
| R0 | 0x08 | | ☐ |
| R1 | 0x04 | | ☐ |
| Z 标志 | 0 | | ☐ |
| C 标志 | 0 | | ☐ |

## 6. 预期结果与异常处置

### 6.1 正常结果

```
R0 = 0x08  （0x05 + 0x03）
R1 = 0x04  （0x03 + 1）
Z = 0，C = 0
```

两项均符合 → ADD、INC 指令功能通过。

### 6.2 ADD 异常

- 若 R0 ≠ 0x08：回到第 3 节核对 0x11 是否写入 0x00；回到第 2 节核对 R0=0x05、R1=0x03 是否真正写入（可对换 R0、R1 初值，再跑一次，看结果是否随初值正确变化）。
- 若 C/Z 标志未更新：核对第 4.5 拍是否同时出现 `LDZ`、`LDC`。

### 6.3 INC 异常（重点）

`pipelined_hardwired_controller.v` 中 INC 分支（`4'b0100`）W1 只给出：

```
DRW SHORT PCINC LIR ABUS LDZ LDC
```

未赋值 `M`、`S`、`CIN`，使用主控制块默认值 `M=0`、`S=4'b0000`、`CIN=1'b0`。

对照 `74ls181.md` 正逻辑算术列，`S=0000` 在 `Cn=H`（无进位输入，对应 ADD 的 CIN=1）时为 `F=A`；`Cn=L`（有进位输入，对应 CIN=0）时为 `F=A-1`。也就是说，**当前 INC 编码可能实现为 A−1 而非 A+1**，与指令格式（Rd←Rd+1）不符。

实测可能出现三种结果之一：

| R1 实测 | 对应 ALU 行为 | 判定 |
| --- | --- | --- |
| 0x04 | A + 1 | 符合语义，INC 编码正确（TEC-8 ALU 对 S=0000,CIN=0 的实际定义为 A+1） |
| 0x02 | A − 1 | INC 实际为 DEC，S/CIN 编码错误，转入 `待确认问题.md` 新增条目 |
| 0x03 | A（传送） | 完全未做算术，ABUS/DRW 路径或 ABUS 输入有异常 |

若出现 0x02 或 0x03，请记录实测值、标志灯状态，并在 `待确认问题.md` 增列一条「INC 指令 ALU 编码」，与 `待确认问题.md` 问题 #6（DEC 编码）一并对照 74181 完整功能表二次确认。
