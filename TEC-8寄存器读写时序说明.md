# TEC-8 寄存器读写与 QD/T3 时序说明

## 1. 基本时序关系

TEC-8 的主时钟 `MF` 频率为 1MHz，周期为 1μs，是整个系统的基准时钟。

执行一条微指令需要 3 个节拍脉冲：

```text
T1、T2、T3
```

TEC-8 采用不定长机器周期，多数操作包含两个机器节拍：

```text
W1、W2
```

少数操作只包含 `W1`，或者包含 `W1、W2、W3`。

在 **DP 朝上，单微指令运行方式** 下：

```text
每按一次 QD，只产生一组 T1、T2、T3
```

也就是说：

```text
第 1 次 QD：当前 W1 产生 T1、T2、T3
第 2 次 QD：当前 W2 产生 T1、T2、T3
第 3 次 QD：下一个 W1 产生 T1、T2、T3
...
```

因此不能简单理解成“一次 QD 同时完成 W1 和 W2”。更准确地说：

> 一次 QD 只执行当前一个 W 节拍，但这个 W 节拍内部包含 T1、T2、T3。

---

## 2. 写寄存器为什么要等 T3

写寄存器属于时序写入操作。

写入 R0-R3 时，控制器需要给出：

```text
SBUS = 1      // 开关数据送 DBUS
DRW  = 1      // 允许写通用寄存器
SEL  = 对应寄存器选择信号
```

但是这些信号有效时，只表示“准备写入”。

真正写入寄存器发生在当前 QD 产生的 `T3` 写入边沿。

所以写寄存器可以理解为：

```text
SEL 选中目标寄存器
SBUS 把开关数据送到总线
DRW 允许寄存器写入
等 T3 边沿到来
寄存器真正更新
```

---

## 3. 写寄存器模式下的顺序

当前 Verilog 中写寄存器模式为：

```verilog
assign WRITE_REG = (SWC_SWB_SWA == 3'b100);
```

复位后：

```text
ST0 = 0
```

写寄存器模式下，控制信号大致如下。

### ST0 = 0 时

第 1 次 QD，当前为 `W1`：

```text
SEL = 0011
SBUS = 1
DRW = 1
```

含义：

```text
选择 R0，准备写 R0
到本次 QD 内部的 T3 边沿时，R0 真正写入
```

第 2 次 QD，当前为 `W2`：

```text
SEL = 0100
SBUS = 1
DRW = 1
```

含义：

```text
选择 R1，准备写 R1
到本次 QD 内部的 T3 边沿时，R1 真正写入
```

第 2 次 QD 结束后，`ST0` 会变为 1。

### ST0 = 1 时

第 3 次 QD，当前为 `W1`：

```text
SEL = 1001
SBUS = 1
DRW = 1
```

含义：

```text
选择 R2，准备写 R2
到本次 QD 内部的 T3 边沿时，R2 真正写入
```

第 4 次 QD，当前为 `W2`：

```text
SEL = 1110
SBUS = 1
DRW = 1
```

含义：

```text
选择 R3，准备写 R3
到本次 QD 内部的 T3 边沿时，R3 真正写入
```

第 4 次 QD 结束后，`ST0` 会回到 0。

因此写寄存器的完整顺序是：

```text
切换到 100 写寄存器模式
按 CLR，ST0 清 0，SEL 灯全灭

第 1 次 QD：W1，SEL=0011，写 R0，T3 后生效
第 2 次 QD：W2，SEL=0100，写 R1，T3 后生效，ST0 变 1
第 3 次 QD：W1，SEL=1001，写 R2，T3 后生效
第 4 次 QD：W2，SEL=1110，写 R3，T3 后生效，ST0 变 0
```

## 4. 读寄存器为什么看起来立即生效

读寄存器和写寄存器不同。

读寄存器只是通过 `SEL` 选择寄存器输出，把寄存器内容接到显示或总线上。

它不需要修改寄存器内容，因此不需要等 T3 写入边沿。

当前 Verilog 中读寄存器模式为：

```verilog
assign READ_REG = (SWC_SWB_SWA == 3'b011);
```

读寄存器代码只给出选择信号：

```verilog
READ_REG: begin
    if (W1) begin
        SELCTL = 1'b1;
        STOP = 1'b1;
        SEL[0] = 1'b1;
    end

    if (W2) begin
        SELCTL = 1'b1;
        STOP = 1'b1;
        SEL[0] = 1'b1;
        SEL[1] = 1'b1;
        SEL[3] = 1'b1;
    end
end
```

所以读寄存器模式下：

```text
第 1 次 QD：W1，SEL = 0001，读 R0/R1
第 2 次 QD：W2，SEL = 1011，读 R2/R3
```

读操作是组合读：

```text
SEL 一选中，寄存器内容就能显示
```

不需要 `DRW`，也不需要等待 T3 修改数据。

## 5. 哪些操作需要等 T3

判断原则：

> 会改变寄存器、标志位或存储单元内容的操作，需要等 T3 或写周期完成。

常见需要等 T3 的操作包括：

```text
DRW   写通用寄存器 R0-R3
LAR   装载地址寄存器 AR
ARINC AR 自增
LPC   装载程序计数器 PC
PCINC PC 自增
PCADD PC 相对加法跳转
LIR   装载指令寄存器 IR
LDZ   更新零标志 Z
LDC   更新进位标志 C
MEMW  写存储器
```

这些信号有效时，通常只表示“允许写入”或“准备更新”，真正状态更新要等时序边沿。

---

## 6. 哪些操作不需要等 T3

不改变状态、只是选择或送总线的操作通常不需要等 T3。

例如：

```text
SELCTL + SEL  选择寄存器读出
SBUS          开关数据送总线
ABUS          ALU 输出送总线
MBUS          存储器输出送总线
M、S、CIN      ALU 运算控制
```

这些是组合逻辑，信号有效后就能看到结果。

---

## 7. 对当前现象的解释

现象：

```text
切换到 100
按 CLR
SEL 灯都不亮
按一次 QD 后，SEL = 0011
```

解释：

```text
100 表示写寄存器模式
CLR 后 ST0 = 0
第一次 QD 进入 W1
SEL = 0011，表示选中 R0
SBUS = 1，开关数据送总线
DRW = 1，允许写寄存器
到本次 QD 内部的 T3 边沿时，R0 真正写入
```

所以第一次 QD 不是没有写 R0，而是：

> 第一次 QD 期间选中了 R0，并在该次 QD 内部的 T3 边沿完成写入。

读寄存器看起来更快，是因为读寄存器不改变内容，只是选择输出：

```text
SEL 一变，数据就能显示
```

而写寄存器必须等：

```text
SEL 选中 + SBUS 有效 + DRW 有效 + T3 边沿
```

---

## 8. 一句话总结

```text
读、选、送总线、ALU 计算：组合操作，不需要等 T3。

写寄存器、写 AR、写 PC、写 IR、更新标志、写存储器：时序操作，需要等 T3 或写周期完成。
```
