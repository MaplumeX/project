# 测试操作注意事项与已知陷阱

本文件记录在 TEC-8 上使用 `pipelined_hardwired_controller.v` 进行功能/综合测试时，容易被忽略但会导致错误结果的**操作陷阱**。这些问题不是控制器代码的 Bug，而是面板操作与时序约束结合后产生的注意事项。

---

## 陷阱 #1：切换模式后必须单独按一次单步锁存 Q

**别名**：「模式锁存」必须独立成 ①a 步

### 问题现象

在综合测试集（老版本）的写存储器初始化里，把「拨 SWC SWB SWA」和「拨 SD」合在同一次单步里：

```
拨 SWC SWB SWA = 001，拨 SD7~SD0 = 0000 0000(0x00)，按一次单步
```

预期结果是 AR ← 0x00，但实际观察到 **AR 灯 = 71H**（下一个待写入字节 0x71 的值），而不是 0x00。程序内存映像被整体破坏。

### 根因

`pipelined_hardwired_controller.v` 中模式开关 Q **不是直接使用**，而是在 T3 下降沿才锁存：

```verilog
always @(negedge T3 or negedge CLR)
begin
    if (!CLR) Q <= 3'b000;
    else case (SWC_SWB_SWA)
        3'b100: Q <= 3'b100;
        ...
    endcase
end
```

也就是说，**一次单步里，「执行控制逻辑」用的是按下之前的旧 Q，「锁存新 Q」发生在这次单步结束的 T3 下降沿**。两者不是同一步生效。

如果把「拨模式开关」和「拨 SD」合在同一次单步里，会发生：

| 实际动作 | 用的 Q | 走的分支 | 实际效果 |
| --- | --- | --- | --- |
| 按下那次单步 | 旧 Q = 000（取指模式） | INS_FETCH, ST0=0, W1 | `SBUS + LPC + STOP + SHORT` → **把 0x00 装进 PC**（不是 AR）|
| 这次单步的 T3 下降沿 | —— | —— | Q 才更新为 001 |
| 下一次单步 | 新 Q = 001 | WR_MEM（但此时 SD 已被拨成下一字节 0x71）| ST0=0 阶段 `SBUS + LAR` → **AR ← 0x71** |

于是 AR 变成了 71H，后续内存映像全部错位。

### 影响

- 写存储器初始化：起始地址装错，内存映像整体错位。
- 读存储器初始化：从错误地址开始读，读出的不是预期数据。
- PC 初始化：起始 PC 装错（不过取指模式自身用旧 Q=000 也是取指分支，副作用更隐蔽）。
- 写寄存器初始化：旧 Q 与新 Q 同为写寄存器分支结构不同，可能写错寄存器号。

### 正确操作方法

**把「锁存模式」和「装地址/写数据」拆成两次单步**：

| 步骤 | 操作 | 作用 |
| --- | --- | --- |
| ①a | 拨 `SWC SWB SWA = 新模式`，按一次单步（SD 任意） | 仅让 T3 下降沿把新模式锁进 Q；用旧 Q 执行的副作用不影响后续 |
| ①b 起 | 拨 `SD7~SD0`，按一次单步 | 此拍起用的已是新 Q，控制信号走正确分支 |

`综合测试集.md` 当前版本的三段初始化（写寄存器/写存储器/PC 初始化）都已按「①a 仅锁存模式 → ①b 装起始地址 → ② 装数据」拆分。

### 与功能测试集的对照

`功能测试集.md` 测试 4.1（写存储器）原本就是两步法：

```
1. 拨 SWC SWB SWA = 001，按一次单步     ← 锁存 Q
2. 拨 SD = 0x00，按一次单步             ← 用新 Q 装 AR
```

这一点是正确的，不要在后续维护时为了「省一步」又合并回去。

## 陷阱 #2：ST0 灯在 TEC-8 面板上不存在

### 问题现象

早期测试集文档大量出现「ST0 灯亮」「ST0 灯灭」作为观察判据。但在 TEC-8 面板上找不到名为 ST0 的指示灯。

### 根因

`pipelined_hardwired_controller.v` 中 ST0 是模块内部的 reg，**没有引出到模块端口**：

```verilog
reg ST0;          // 仅模块内部状态位，无 output 声明
wire SST0;        // 仅模块内部连线，无 output 声明
reg [2:0] Q;      // 仅模块内部，也无 output 声明
```

因此它们无法连接到 EPM7128 的物理引脚，面板上自然没有对应的灯。

### 影响

直接用「ST0 灯」无法判断当前阶段，必须改用面板上可见的灯组合。

### 解决方法：用面板信号灯组合判断 ST0 阶段

| 模式 | ST0=0 阶段特征灯 | ST0=1 阶段特征灯 |
| --- | --- | --- |
| 写存储器 001 | **`LAR` 亮**、`MEMW` 灭、`ARINC` 灭 | `LAR` 灭、**`MEMW` 亮**、**`ARINC` 亮** |
| 读存储器 010 | **`SBUS` 亮、`LAR` 亮** | **`MBUS` 亮、`ARINC` 亮**、`SBUS` 灭、`LAR` 灭 |
| 写寄存器 100 | SEL=`0011`/`0100`，`DRW` 亮 | SEL=`1001`/`1110`，`DRW` 亮（写寄存器全程 DRW 都亮）|

`综合测试集.md` 1.1 节已给出此速查表，所有初始化步骤里判断阶段都已改用这些可见灯组合。
