 ● 为什么"T3 下降沿才是真正把数据打进寄存器的时刻"

   这是 TEC-8 数据通路本身的硬件设计决定的，不是控制器代码的选择。需要区分两件事：控制信号什么时候有效，和寄存器什么时候
   真正改写。

   1. 控制信号只是"打开门"，真正写入还需要一个时钟边沿

   一条写入操作（比如写寄存器 R0）需要三件东西同时具备：

   ```
     SEL 选中 R0      （选择哪个寄存器）
     SBUS = 1         （开关数据送上 DBUS）
     DRW  = 1         （允许写寄存器）
     T3 下降沿        （触发寄存器真正改写）
   ```

   SEL、SBUS、DRW 是组合逻辑产生的电平信号——它们只是把数据"摆到寄存器门口、把写使能打开"。通用寄存器、AR、PC、IR 这些都
   是边沿触发的时序器件，光有写使能还不行，必须等一个时钟跳变沿，数据才真正被锁进去。

   TEC-8 把这个统一写入边沿定在了 T3 的下降沿。

   2. 为什么统一用 T3 而不是 T1/T2

   一个机器节拍 T1 → T2 → T3 内，各段时间的职责是分配好的：

   - T1、T2：数据通路稳定阶段。ALU 运算、总线传输、存储器读出等组合过程在 T1/T2 高电平期间完成并稳定。此时控制信号已经驱
     动好数据，但还没打进去。
   - T3：收尾写入阶段。等到 T1/T2 期间数据已经稳定在总线上，T3 给出写入边沿，一次性把"已经稳定的数据"锁进目的寄存器。

   如果写入边沿放在 T1 上升沿（周期开始），那时上一拍的总线数据可能还没建立/还没稳定，会锁进垃圾值；放在 T2 中间又会切断
   T1→T3 的稳定窗口。所以放在最后一段 T3，并取其下降沿，保证：

   - 下降沿到来前（T3 高电平期间）：数据和写使能已经稳定；
   - 下降沿时刻：干净地锁存一次；
   - 下降沿之后（进入下个 T1）：寄存器已更新，可供下一条指令使用。

   3. 控制器确实也是这么用它来对齐状态切换的

   正因为"T3 下降沿 = 本拍写入收尾"，控制器把需要"在本次操作完成、下次操作开始前"更新的状态，全部挂在同一个 negedge T3
   上：

   ```verilog
     always @(negedge T3 or negedge CLR) Q <= ...;    // 模式锁存
     always @(negedge T3 or negedge CLR) ST0 <= ...;  // 阶段状态
   ```

   而像读寄存器那种纯组合操作（SELCTL + SEL 选通输出，不写内容）就不需要等 T3——TEC-8寄存器读写时序说明.md 第 4 节里"读寄
   存器为什么看起来立即生效"说的就是这个：读只是选通输出，SEL 一变数据就显示出来，根本没用到 T3 写沿。

   4. 这就是"写寄存器要等 T3"现象的根因

   TEC-8寄存器读写时序说明.md 里反复强调的那条规律：

   │ 会改变寄存器、标志位或存储单元内容的操作，需要等 T3；不改变状态、只是选择或送总线的操作，不需要等 T3。

   根因就是上面这套"组合门控信号 + 统一 T3 下降沿写入"的数据通路设计。所有写类控制信号（ DRW / LAR / LPC / LIR / LDZ /
   LDC / MEMW / ARINC / PCINC / PCADD ）本质都是"写使能门"，它们拉高只代表"允许写"，真正改写发生在当前 QD 产生的那组
   T1/T2/T3 中的 T3 下降沿。

   一句话

   因为 TEC-8 的寄存器、AR、PC、IR、存储器都是边沿触发的时序器件，而 TEC-8 把所有写入操作的统一时钟边沿规定在 T3 下降沿
   ——在 T1/T2 期间用控制信号把数据稳定到门口，T3↓ 时刻才真正锁存进去。控制信号（SEL/DRW/LAR…）只是打开门的组合电平，T3
   下降沿才是"关门落锁"的动作
