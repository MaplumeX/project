# TEC-8 的 T3 上升沿 / 下降沿分别干什么

## 0. 一句话纠正

> **数据通路写入(R0–R3、AR、PC、IR、Z/C、存储器)发生在 T3 上升沿;控制器内部状态(Q、ST0)在 T3 下降沿更新。**
>
> 早期版本的本文曾把"数据写入"也归到 T3 下降沿,这是错的。数据通路各寄存器/标志/存储器的写入边沿是 **上升沿**,下降沿是控制器自身状态切换用的,二者职责不同、故意错开。

依据:
- 参考控制器原件(`.git/lost-found/other/` 下的 TEC-8 控制器注释)逐条注明 `DRW / LAR / LPC / LDC / LDZ / ARINC / PCINC` 均为"在 T3 上升沿"生效。
- 本仓库 `pipelined_hardwired_controller.v` 中控制信号为组合(`assign`),其有效只需在 T3 上升沿前稳定;而 `Q`、`ST0` 显式用 `always @(negedge T3)`。

---

## 1. 区分两件事:控制信号何时有效 vs 寄存器何时真正改写

一条写寄存器操作(以写 R0 为例)需要四件东西同时具备:

```
SEL 选中 R0      （选择哪个寄存器）
SBUS = 1         （开关数据送上 DBUS）
DRW  = 1         （允许写寄存器）
T3 上升沿        （触发寄存器真正改写）
```

`SEL / SBUS / DRW` 是组合逻辑产生的**电平信号**:它们只是把数据"摆到寄存器门口、把写使能打开"。R0–R3、AR、PC、IR 这些都是**边沿触发的时序器件**,光有写使能还不够,必须等一个时钟跳变沿,数据才真正被锁进去——TEC-8 把这个统一写入边沿定在 **T3 上升沿**。

所以"等 T3"指的是**等 T3 上升沿把数据锁进去**,不是等下降沿。

## 2. 为什么统一用 T3(而不是 T1/T2)

一个机器节拍 T1 → T2 → T3 内,各段职责是分配好的:

- **T1、T2**:数据通路稳定阶段。ALU 运算、总线传输、存储器读出等组合过程在 T1/T2 高电平期间完成并稳定。此时控制信号已驱动好数据,但还没打进去。
- **T3**:收尾写入阶段。等到 T1/T2 期间数据已稳定在总线上,T3 上升沿一次性把"已经稳定的数据"锁进目的寄存器。

把写入边沿放在最后一段的 T3,并取上升沿,保证:

- 上升沿到来前(T1/T2 + T3 低电平期间):数据和写使能已经稳定;
- 上升沿时刻:干净地锁存一次;
- 上升沿之后(T3 高电平期间到下一拍):寄存器已更新,可供后续使用。

如果放在 T1 上升沿,上一拍的总线数据可能还没建立/稳定,会锁进垃圾值;放 T2 中间又会切断 T1→T3 的稳定窗口。

## 3. Q 和 ST0 是什么:它们属于"控制器",不属于"数据通路"

TEC-8 里有两类寄存器,职责完全不同:

| 类别 | 例子 | 干什么 | 在哪个沿更新 |
|---|---|---|---|
| **数据通路寄存器** | R0–R3、AR、PC、IR、Z、C、存储器 | 存"数据/地址/指令" | **T3 上升沿** |
| **控制器内部状态** | `Q`、`ST0` | 存"当前在哪个模式、哪个阶段",用来决定下一拍该给哪些控制信号 | **T3 下降沿** |

`Q`、`ST0` **不存任何运算数据**,它们是控制器用来决定"这一拍该拉高哪些控制信号"的状态变量。

### Q(模式锁存)

```verilog
always @(negedge T3 or negedge CLR)
    case (SWC_SWB_SWA)
        3'b100: Q<=3'b100;  // 写寄存器模式
        3'b011: Q<=3'b011;  // 读寄存器模式
        3'b000: Q<=3'b000;  // 取指模式
        ...
    endcase
```

`Q` 就是把面板模式开关 `SWC_SWB_SWA` **锁存一份稳定值**。不直接用原始开关,是为了**避免机械开关抖动直接影响控制输出**。后续所有译码都基于这个稳定的 `Q`(如 `WRITE_REG = (Q==3'b100)`、`SST0` 也用 `Q`)。

详见 `测试操作注意事项.md` 陷阱 #1:切换模式后必须单独按一次单步锁存 Q——正是因为"执行控制逻辑用的是旧 Q,锁存新 Q 发生在这次单步结束的 T3 下降沿"。

### ST0(阶段状态)

`ST0` 是一比特阶段标志,标记当前流程走到"前阶段"还是"后阶段"。以写寄存器模式为例:

- 第 1 次 QD(W1),`ST0=0` → 写 R0
- 第 2 次 QD(W2),`ST0=0` → 写 R1,本次 QD 的下降沿把 `ST0` 翻成 1
- 第 3 次 QD(W1),`ST0=1` → 写 R2
- 第 4 次 QD(W2),`ST0=1` → 写 R3,本次 QD 的下降沿把 `ST0` 翻回 0

`ST0` 让控制器能用同一组 W1/W2 节拍区分出多个阶段,决定"同一节拍但处于前/后阶段时,选不同寄存器、走不同分支"。

一句话:**Q 回答"现在在哪种工作模式",ST0 回答"流程走到前半段还是后半段"。它们都是控制器自身的状态,用来决定本拍该发哪些控制信号。**

## 4. 为什么 Q / ST0 放 T3 下降沿:和上升沿的数据写入错开

核心原因:**和数据通路在 T3 上升沿的写入,在时间上错开,形成一个"先做事、后切状态"的干净分界。**

把一个 W 节拍的 T3 脉冲展开:

```
T3:  ___|‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾|___
          ↑ 上升沿        ↓ 下降沿
          写数据通路        换控制器状态
          (用旧 Q/ST0)     (为下一拍准备)
```

- **T3 上升沿**:R0–R3、AR、PC、IR 等数据通路寄存器在这一刻真正写入数据。此刻用的**还是旧的 Q、ST0**——也就是"这一拍该做什么"在整个 T3 高电平期间保持稳定,数据干净地写进去。
- **T3 下降沿**:`Q`、`ST0` 在这一刻才更新。也就是等"这一拍的数据都写完了",才切换控制器的内部状态,为**下一拍**准备好新的模式/阶段。

如果让 Q/ST0 也在上升沿更新,就会出问题:上升沿那一瞬间,数据通路正按"旧的 Q/ST0"锁存数据,而 Q/ST0 同时变成新值、马上影响后面的组合译码——控制和数据写入抢同一时刻,容易乱。

所以下降沿给 Q/ST0 的语义是:**"这一拍的处理到此收尾,在进入下一拍之前,把模式/阶段标志更新好。"** 它不是数据写沿,而是**控制状态的交接沿**。

## 5. 三种"沿相关动作"对照

| 沿 | 谁在动 | 动作性质 |
|---|---|---|
| T3 上升沿 | R0–R3、AR、PC、IR、Z/C、存储器 | **数据写入**(数据通路内容更新) |
| T3 高电平期间 | `PCINC`、`LIR`(由 `posedge T3` 寄存器锁存) | 控制信号已稳定,等待被数据通路响应 |
| T3 下降沿 | `Q`、`ST0` | **控制状态切换**(为下一拍准备模式/阶段) |

> 关于 `PCINC`/`LIR`:本控制器把它们从组合逻辑改为 `posedge T3` 寄存器锁存,是为了让两个信号在 PC/IR 响应前严格同步、消除"吞指令"反序触发。PC/IR 究竟在上升沿还是下降沿"响应",是一个与具体实验台有关的时序假设,详见 `流水线吞指令问题记录.md` 的 ⚠️ 实测验证前提,不在本文结论范围内。

## 6. 读/组合操作为什么不等 T3

读寄存器等纯组合操作(`SELCTL + SEL` 选通输出,不写内容)不需要等 T3——`SEL` 一变数据就显示出来,不触发任何写入。这是因为它们不改状态,只是选通输出。详见 `TEC-8寄存器读写时序说明.md` 第 4、6 节。

判断原则:

> 会改变寄存器、标志位或存储单元内容的操作,等 T3 **上升沿**;不改变状态、只是选择或送总线的操作,不需要。

## 7. 一句话总结

```
T3 上升沿  →  数据通路写入(R0–R3/AR/PC/IR/Z/C/存储器):关门落锁,真正改写数据。
T3 下降沿  →  控制器状态切换(Q/ST0):这一拍做完,为下一拍换好模式/阶段。
控制信号(SEL/DRW/LAR…)  →  组合电平,只是"开门",在上升沿前稳定即可。
```
