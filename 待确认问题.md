## 问题 #5：NOT 指令缺少 `S` 编码

**位置**：`pipelined_hardwired_controller.v`，指令 `4'b1101`（NOT）的 W1 分支。

**当前代码**：

```verilog
4'b1101: begin // NOT：执行按位取反并写回寄存器。
    if (W1) begin // 长周期第 1 拍：执行取反，结果经 ABUS 写回寄存器。
        DRW = 1'b1;
        ABUS = 1'b1;
        LDZ = 1'b1;
        M = 1'b1;
        LONG = 1'b1;
        // 注意：未对 S 赋值，S 保持默认 4'b0000
    end
    ...
end
```

**问题描述**：

NOT 是逻辑运算（`M = 1`），按位取反需要 ALU 提供专门的"取反"功能码。
当前代码只设了 `M = 1`，没有设置 `S`，于是 `S` 保持组合逻辑块开头的默认值
`4'b0000`。`S = 0000` 在逻辑模式下对应的运算是否为"按位取反"，需查 ALU 功能表
确认。若 `0000` 对应的是"直通/传送"而非"取反"，则 NOT 指令实际不会执行取反，
只相当于一次寄存器自我传送。

**对照参考**：

| 指令 | M | S | 期望运算 |
| --- | --- | --- | --- |
| AND | 1 | 1011 | 按位与 |
| NOT | 1 | (未设,=0000) | 按位取反 ? |

同类逻辑运算（AND）都明确给出了 `S` 编码，唯独 NOT 漏设，可疑程度较高。

**确认方法**：

1. 查阅 TEC-8 ALU 功能表，找到 `M=1` 时各 `S` 编码对应的运算；
2. 确认其中是否存在"按位取反"运算及其对应的 `S` 值；
3. 将该 `S` 值补入 NOT 指令的 W1 分支。

**若确认为 Bug 的修复示例**（假设取反对应 `S = 1100`，以实际功能表为准）：

```verilog
M = 1'b1;
S = 4'b1100;   // 以 TEC-8 ALU 功能表给出的取反编码为准
```

## 问题 #6：DEC 指令 ALU 编码 `S / CIN` 可疑

**位置**：`pipelined_hardwired_controller.v`，指令 `4'b1111`（DEC）的 W1 分支。

**当前代码**：

```verilog
4'b1111: begin // DEC：执行自减并写回寄存器，同时更新 Z/C。
    if (W1) begin // 长周期第 1 拍：执行自减，结果写回寄存器并更新标志位。
        DRW = 1'b1;
        CIN = 1'b1;
        ABUS = 1'b1;
        LDZ = 1'b1;
        LDC = 1'b1;
        S = 4'b1111;
        LONG = 1'b1;
    end
    ...
end
```

**问题描述**：

DEC（自减，即 `A - 1`）的常用实现思路有两种：

1. **减法思路**：`A - 1 = A + (-1 的补码) = A + 1111 + 0`，使用减法功能码
   （参考 SUB：`S = 0110`），`CIN = 0`。
2. **加法思路**：`A - 1 = A + 1111 + 1 = A + (1111 + 1) 溢出回 0000`，使用
   加法功能码（参考 ADD：`S = 1001`），`CIN = 1`。

当前代码使用 `S = 1111` + `CIN = 1`，既不是 SUB 的 `0110`，也不是 ADD 的 `1001`，
而是用了与 ST/JMP 的"传送/输出地址"相同的 `1111` 编码。`S = 1111` 在 `CIN = 1`
时是否恰好等于 `A - 1`，取决于该 ALU 对 `1111` 功能码的具体定义，需查表确认。

**对照参考**：

| 指令 | 运算 | S | CIN |
| --- | --- | --- | --- |
| ADD | A + B | 1001 | 1 |
| SUB | A - B | 0110 | 0 |
| INC | A + 1 | 0000 | (未设,=0) |
| DEC | A - 1 | 1111 | 1 ← 可疑 |

INC（自增）用的是 `S = 0000` + 默认 `CIN = 0`（即 `A + 0`，靠 ALU 内部
`+1` 通路实现自增，见流程图 INC 项 `S = 0000`）。DEC 与 INC 是对称操作，
编码风格却完全不同，可疑。

**确认方法**：

1. 查阅 TEC-8 ALU 功能表，确认 `S = 1111, CIN = 1`（算术模式）的运算结果；
2. 若结果不是 `A - 1`，则需改为正确的减法编码，例如：
   - `S = 0110`（减法）+ `CIN = 0`，或
   - `S = 1001`（加法）+ `CIN = 1`（让 `A + 1111 + 1` 溢出回 0 实现 `-1`），
     以功能表实际定义为准。
3. 同步核对 `LDC` 是否合理：若采用加法思路，进位标志语义为"无符号溢出"，
   与 SUB 的借位语义不同，需确认实验要求是否接受。
