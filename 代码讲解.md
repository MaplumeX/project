# ver1 硬布线控制器代码讲解

本文档讲解 `ver1.v` 的整体结构、输入输出信号含义、状态控制方式，以及各工作模式和指令的执行流程。

## 1. 模块作用

`ver1` 是一个 TEC-8 风格的硬布线控制器模块。

它根据以下信息产生控制信号：

- 面板模式开关 `SWC_SWB_SWA`
- 指令寄存器高 4 位 `IR7_IR4`
- 时序节拍 `W1`、`W2`、`W3`
- 时序信号 `T3`
- 复位信号 `CLR`
- 标志位 `C`、`Z`

输出信号用于控制数据通路，例如寄存器写入、ALU 输出、存储器读写、PC 更新和 IR 装载等。

## 2. 整体结构

代码主要分为四部分：

1. **端口声明**：定义输入输出控制信号。
2. **模式锁存与译码**：在 `T3` 下降沿采样 `SWC_SWB_SWA`，并译码成具体工作模式。
3. **状态位 `ST0` 控制**：用 `ST0` 区分当前操作处于第一阶段还是第二阶段。
4. **组合控制逻辑**：根据模式、节拍、指令和标志位产生具体控制信号。

## 3. 工作模式

`SWC_SWB_SWA` 用于选择控制器工作模式。

| 开关值 | 模式 | 代码中的译码信号 |
| --- | --- | --- |
| `100` | 写寄存器 | `WRITE_REG` |
| `011` | 读寄存器 | `READ_REG` |
| `000` | 取指/执行 | `INS_FETCH` |
| `010` | 读存储器 | `READ_MEM` |
| `001` | 写存储器 | `WRITE_MEM` |

在 `always @(negedge T3)` 中，模块会把当前开关值锁存到寄存器 `Q`。后续所有模式判断都基于 `Q`，这样可以避免开关输入直接影响组合控制逻辑。

## 4. ST0 状态位

`ST0` 是控制器内部阶段标志。

- `ST0 = 0`：通常表示初始阶段，例如装载地址、装载 PC 或写入第一组数据。
- `ST0 = 1`：通常表示后续阶段，例如读写存储器数据或执行指令。

`SST0` 是 `ST0` 的置位条件。不同模式进入下一阶段所需节拍不同：

| 模式 | 置位条件 |
| --- | --- |
| 写寄存器 | `W2` |
| 读存储器 | `W1` |
| 写存储器 | `W1` |
| 取指/执行 | `W2` |

当 `CLR` 为低电平时，`ST0` 被复位为 `0`。

## 5. 输出信号初始化

主组合逻辑位于：

```verilog
always @(*) begin
    ...
end
```

每次进入组合逻辑时，代码会先把所有输出信号清零，再根据当前模式、节拍和指令选择性拉高需要的控制信号。

这种写法有两个作用：

1. 避免遗漏分支导致综合出锁存器。
2. 让每个控制信号只在需要的微操作中有效。

## 6. 手动操作模式

### 6.1 写寄存器

写寄存器模式由 `WRITE_REG` 控制。

- `W1`：通过 `SBUS` 把开关输入送到总线，并写入指定寄存器。
- `W2`：继续完成另一组寄存器写入，同时根据 `ST0` 选择不同寄存器。

关键控制信号包括：

- `SBUS`：开关数据送总线。
- `SELCTL`：启用寄存器选择控制。
- `DRW`：允许写寄存器。
- `STOP`：暂停自动运行，配合手动操作。
- `SEL`：选择目标寄存器。

### 6.2 读寄存器

读寄存器模式由 `READ_REG` 控制。

该模式只选择寄存器输出，不写寄存器，因此主要使用：

- `SELCTL`
- `SEL`
- `STOP`

### 6.3 读存储器

读存储器模式由 `READ_MEM` 控制。

- `ST0 = 0` 时：通过 `SBUS` 输入地址，并用 `LAR` 装入地址寄存器。
- `ST0 = 1` 时：通过 `MBUS` 读出存储器数据，同时 `ARINC` 使地址寄存器自增。

### 6.4 写存储器

写存储器模式由 `WRITE_MEM` 控制。

- `ST0 = 0` 时：通过 `SBUS` 输入地址，并装入地址寄存器。
- `ST0 = 1` 时：通过 `SBUS` 输入数据，`MEMW` 写入存储器，同时 `ARINC` 使地址寄存器自增。

## 7. 取指与执行模式

取指/执行模式由 `INS_FETCH` 控制。

### 7.1 `ST0 = 0`：初始化 PC

当 `ST0 = 0` 时，控制器还没有进入正式指令执行阶段。

- `W1`：输出 `STOP`，等待手动输入。
- `W2`：通过 `SBUS` 把开关输入装入 PC，`LPC` 有效。

### 7.2 `ST0 = 1`：指令译码执行

当 `ST0 = 1` 时，控制器根据 `IR7_IR4` 执行不同指令。

## 8. 指令译码表

| `IR7_IR4` | 指令 | 功能 |
| --- | --- | --- |
| `0000` | `NOP` | 空操作，取下一条指令 |
| `0001` | `ADD` | 加法，结果写回寄存器 |
| `0010` | `SUB` | 减法，结果写回寄存器 |
| `0011` | `AND` | 按位与，结果写回寄存器 |
| `0100` | `INC` | 自增，结果写回寄存器 |
| `0101` | `LD` | 从存储器读数据到寄存器 |
| `0110` | `ST` | 将寄存器数据写入存储器 |
| `0111` | `JC` | C 标志为 1 时跳转 |
| `1000` | `JZ` | Z 标志为 1 时跳转 |
| `1001` | `JMP` | 无条件跳转 |
| `1010` | `OUT` | 输出寄存器数据 |
| `1011` | `MOV` | 寄存器间传送 |
| `1100` | `CMP` | 比较，只更新标志位 |
| `1101` | `NOT` | 按位取反 |
| `1110` | `STP` | 停机 |
| `1111` | `DEC` | 自减，结果写回寄存器 |

## 9. 典型指令流程

### 9.1 单节拍指令

`NOP`、`ADD`、`SUB`、`AND`、`INC` 等指令通常在 `W1` 完成。

它们通常会同时完成两类动作：

1. 执行当前指令对应的 ALU 或控制操作。
2. 通过 `PCINC` 和 `LIR` 取下一条指令。

例如 `ADD` 在 `W1` 中会使：

- `ABUS = 1`：ALU 输出送总线。
- `DRW = 1`：结果写回寄存器。
- `LDZ = 1`：更新零标志。
- `PCINC = 1`：PC 加 1。
- `LIR = 1`：装载下一条指令。

### 9.2 多节拍指令

`LD`、`ST`、`JC`、`JZ`、`JMP` 等指令需要两个节拍。

以 `LD` 为例：

- `W1`：通过 ALU/ABUS 计算或输出地址，并用 `LAR` 装入地址寄存器。
- `W2`：通过 `MBUS` 读出存储器数据，并用 `DRW` 写回寄存器，同时取下一条指令。

### 9.3 条件跳转指令

`JC` 和 `JZ` 会根据标志位决定是否跳转。

- 条件不满足：设置 `SHORT`，直接 `PCINC` 并 `LIR`。
- 条件满足：设置 `LONG`，先执行 `PCADD`，再在 `W2` 中取下一条指令。

## 10. 常见控制信号含义

| 信号 | 含义 |
| --- | --- |
| `SELCTL` | 启用寄存器选择控制 |
| `ABUS` | ALU 输出送总线 |
| `SBUS` | 开关输入送总线 |
| `MBUS` | 存储器输出送总线 |
| `M` | ALU 运算模式控制 |
| `CIN` | ALU 进位输入 |
| `DRW` | 寄存器写使能 |
| `LDZ` | 写入零标志 |
| `LDC` | 写入进位标志 |
| `MEMW` | 存储器写使能 |
| `ARINC` | 地址寄存器自增 |
| `PCINC` | PC 自增 |
| `PCADD` | PC 相对加法跳转 |
| `LPC` | 装载 PC |
| `LAR` | 装载地址寄存器 |
| `LIR` | 装载指令寄存器 |
| `STOP` | 停机或等待手动操作 |
| `SHORT` | 当前指令为短周期 |
| `LONG` | 当前指令为长周期 |
| `S` | ALU 功能选择 |
| `SEL` | 寄存器选择 |

## 11. 阅读代码的方法

阅读 `ver1.v` 时可以按以下顺序理解：

1. 先看端口列表，明确每个输入输出信号的方向和作用。
2. 再看 `Q` 的模式锁存逻辑，确定当前处于哪种工作模式。
3. 查看 `ST0` 和 `SST0`，理解操作如何从第一阶段进入第二阶段。
4. 进入 `always @(*)`，按 `WRITE_REG`、`READ_REG`、`READ_MEM`、`WRITE_MEM`、`INS_FETCH` 五个分支阅读。
5. 在 `INS_FETCH` 分支中，根据 `IR7_IR4` 对照指令表查看每条指令的微操作。

两者之间的对应关系如下：

- 流程图中的 `ST0` 分支对应代码中的 `ST0` 状态位。
- 流程图中的 `W1`、`W2` 条件对应代码中各分支内的 `if (W1)` 和 `if (W2)`。
- 流程图中的控制信号名称基本直接对应代码中的输出端口。
